# Sprint 1 Plan

## Sprint Goal

Deliver a functional minimum viable product that allows Dungeon Masters to create, view, edit, and delete NPCs with persistent storage in a hosted database.

This sprint establishes LoreSmith’s core vertical slice and validates external database connectivity, data persistence, and end-to-end CRUD capability.

---

## Sprint Scope

Sprint 1 focused on delivering the foundational campaign management capability: persistent NPC management.

The sprint prioritized establishing the application's basic architecture, validating database connectivity, and implementing the first full CRUD workflow.

---

## Committed User Stories

- Persist Data in Hosted Database (8)
- Create NPC (3)
- View NPC List (2)
- Edit NPC (3)
- Delete NPC (2)

**Total Commitment:** 18 Story Points

---

## Expected Outcome

By the end of Sprint 1 the application should support:

- Creating NPC records
- Viewing a list of NPCs
- Editing existing NPC records
- Deleting NPC records
- Persistent storage of NPC data in a hosted database

This functionality represents the first usable vertical slice of the LoreSmith campaign management tool.

---

## Notes

Sprint 1 intentionally avoided introducing additional entities (such as factions or session logs) in order to focus on validating the core persistence and CRUD architecture.
